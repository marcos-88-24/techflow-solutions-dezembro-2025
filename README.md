# techflow-solutions-dezembro-2025
## Levantamento de Requesitos

Antes do criação das issues, foi realizada uma entrevista
inicial com cliente para entender as necessidades do projeto.

### Requisitos Funcionais

- RF01 – O sistema deve permitir cadastrar produtos, informando nome, preço e quantidade em estoque.
- RF02 – O sistema deve permitir visualizar todos os produtos cadastrados em forma de lista.
- RF03 – O sistema deve permitir atualizar as informações de um produto já cadastrado.
- RF04 – O sistema deve permitir excluir produtos do estoque.
- RF05 – O sistema deve permitir que o usuário selecione produtos por meio de uma lista numerada.
- RF06 – O sistema deve apresentar um menu principal com as opções disponíveis.
- RF07 – O sistema deve permitir encerrar a execução quando o usuário escolher a opção de sair.


### Requisitos Não Funcionais

- RNF01 – O sistema deve possuir interface simples e de fácil entendimento, adequada para usuários leigos.
- RNF02 – O sistema deve ser desenvolvido na linguagem Python.
- RNF03 – O sistema deve executar de forma contínua, até que o usuário solicite o encerramento.
- RNF04 – O sistema deve organizar as informações de forma clara, facilitando a leitura dos dados.
- RNF05 – O sistema deve ser compatível com sistemas Windows e Linux.

### Escopo do Projeto
- Cadastro de produtos
O sistema permite cadastrar produtos informando nome, preço e quantidade disponível em estoque.
- Atualização de produtos
É possível alterar as informações de produtos já cadastrados, como nome, preço e quantidade.
- Exclusão de produtos
O usuário pode remover produtos do estoque quando eles não forem mais necessários.
- Visualização do estoque
O sistema permite visualizar todos os produtos cadastrados em forma de lista organizada.
- Menu interativo em modo texto
O sistema funciona no terminal, apresentando um menu simples para navegação entre as opções disponíveis.


### Tecnologias Utilizadas

- Python 3 Linguagem de programação utilizada 
- Terminal / Console, Ambiente onde o sistema é executado.
- Biblioteca padrão os (Python)
- Sistema Operacional (Windows ou Linux)

### Gestão do projeto

- Metodologia:  Kanban via GitHub Projects.

- Desenvolvimento issues #1 a #5 para criação, fechamento ao implantar
